Build a lightweight client portal for a trades/service business website. This is a simple web app where the business owner's clients can log in and request changes to their website.

Auth:

Email/password login and registration
JWT cookie sessions (7-day expiry)
Protected routes redirect to /login if not authenticated
Pages:

/login — login form with email + password. Link to register.

/register — registration form with business name, email, password.

/ (dashboard) — simple welcome screen after login. Shows their business name. Can expand later.

/site-settings — this is the main feature. Two tabs:

Quick Updates tab — a structured form with labeled fields:

Business Name
Phone Number
About Section Text (textarea)
Services List (textarea — "list each service on a new line")
Pricing Notes (textarea)
Photo Instructions (textarea — "describe what photos you want added/replaced, or paste a Google Drive/Dropbox link")
On submit: save to database, show success message: "Got it! We'll apply your changes within 24–48 hours."
Custom Request tab — a single large textarea with placeholder: "Describe any change you'd like — move a section, add a new page, change the color scheme, etc. The more detail the better." Same submit/success behavior.

Below both tabs: Request History — a list of all past submissions for this user, showing date, type (Quick Update / Custom Request), and a status badge: Pending / In Progress / Completed.

Database:

users table: id, email (unique), password (bcrypt hashed), business_name, created_at
site_change_requests table: id, user_id (FK), request_type (structured/custom), business_name, phone, about_text, services_text, pricing_notes, photo_notes, prompt_text, status (pending/in_progress/completed), admin_notes, created_at, updated_at
Admin API (no UI needed — just REST endpoints protected by an x-admin-api-key header):

GET /api/admin/site-changes — list all requests across all users, newest first
PATCH /api/admin/site-changes/:id — update status and/or add admin notes
Tech: React + Vite + Tailwind + shadcn/ui on the frontend. Express + PostgreSQL + Drizzle ORM on the backend. Match the existing site's blue color scheme.

Important: This is multi-tenant — every user only ever sees their own requests. All data queries must filter by the logged-in user's ID.
